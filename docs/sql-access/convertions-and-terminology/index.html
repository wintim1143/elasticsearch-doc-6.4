公约和术语

公约和术语

为清晰起见，建立某些重要单词背后的意义是很重要的，因为相同的措辞可能会根据一个人对SQL与Elasticsearch的熟悉程度向不同的读者传达不同的含义。

note:

在尝试完成文档时，假设读者对Elasticsearch和/或SQL有基础的了解。如果不是这样的情况，请继续阅读文档，但是记录笔记并通过主要的Elasticsearch文档和大量开放的SQL资料来查找不清楚的主题（有太多的优秀的质料可供枚举）。

作为一个通用规则，Elasticsearch SQL作为名称表示为Elasticsearch提供一个sql接口。因此，只要有可能，首先会遵守sql公约和术语。但是，由于后备引擎本身就是Elasticsearch,因此为其创建的Elasticsearch SQL,会出现很多在SQL没有，或映射不上的概念。最后但同样重要的是，Elasticsearch SQL试图去准寻最不惊讶原则（principle of least suprise），尽管在这个世界上所有的事情都是相对的。
